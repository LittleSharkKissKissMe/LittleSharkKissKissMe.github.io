质数问题

质数问题的通病是跑得太慢，经常会出现时间超限的情况
下面介绍几种相对高效的方法

1.一般方法
核心代码：

#include<iostream>
using namespace std;
bool isComp[100000005];

int main (){
 int n,ans=0;
 cin>>n;

 isComp[1]=1;  //1的特判

 for(int i=2;i*i<=n;i++){ 

     if(isComp[i]==0)
         for(int j=i*i;j<=n;j+=i)
     	    isComp[j]=1;

 }


 for(int i=1;i<=n;i++){

  if(isComp[i]==0)
   ans++;

 }

 cout<<ans<<endl;

 return 0;  

}

（来自洛谷题解.luoyoucheng）

也就是从2开始把它自身的所有倍数标记为非质数，那么剩下的就是质数

2.埃式筛法

核心代码：

bitset < maxn > pri;//预处理，0为素数，1为合数

for(int i=2; i<=N/i; ++i){  //从根号n开始筛选

    if(!pri[i])  //如果i是质数
        for(int j=i*i; j<=N; j+=i) pri[j]=1;  //从它的平方开始+i筛选

}

（来自BV1hR4y1u7e1）

3.欧拉筛法

优化了埃式筛法中重复筛选的部分，复杂度接近线性

相比上面的算法，欧拉筛法理解起来没那么容易
首先，它从2到n（i）遍历，对于遍历的每个数，假如是素数，就把它加入到素数表
接下执行子循环，从1到已知素数的个数（j）遍历，把所有 i和已知素数（假设为a）的乘积 筛掉
然后在这个子循环里，关键的下一句话是：如果i是已知素数（a）的倍数，退出子循环

这是为什么？
这句话的作用，是保证每个合数只被筛一遍，即防止重复筛

举个例子，比如说12（223）
12=43
12=62
当i=4时，12会被43筛掉，关键语句不触发，因为4不是3的倍数
当i=6时则不会被62筛掉。因为6是2的倍数，关键语句执行，直接退出了

再举个例子，30（235）
30=65 （6包含2，提前break）
30=103 (10包含2，提前break)
30=15*2 (15最小包含3，但是3在2（另一个因数）后面，所以先筛了30再break)

不难发现，起筛选作用的i是这个合数 最大的 非素数因数（假设为x）
这里是15
另一个因数就是这个数的最小质因数（y）
这里是2

只需证明x的最小质因数比y小，也就是证明只有这种情况下不会提前break

非常容易证，因为y是整个数的最小质因数，所以肯定是最小的，或者至少和x一样小

证明完毕，虽然根本没用（

记住模板得了：
核心代码：

#include<bits/stdc++.h>
#define maxb 1000000
using namespace std;
bool pan[100000100];  //判断是不是素数的数组
int prime[1000000],num,n,m;  //prime数组存得到的质数，从小到大存


void su(){

    memset(pan,true,sizeof(pan));
    pan[0]=false;
    pan[1]=false;

    for(int i=2;i<=n;i++){

        if(pan[i]){  //如果是素数，把它存到数组里，同时个数+1
            prime[++num]=i;
        }


        for(int j=1;prime[j]*i<=n && j<=num;j++){

            pan[i*prime[j]]=false;

            if(i%prime[j]==0){  //算法的核心
                break;

            }
        }
    }        
}
