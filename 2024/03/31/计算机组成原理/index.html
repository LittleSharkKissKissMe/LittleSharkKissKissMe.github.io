第二章：运算方法和运算器

2.1 数字与文字的表示方法

进制转换：

    十进制转非十进制

    整数部分较为简单。小数部分采用乘基取整法。

    非十进制转十进制

    整数部分较为简单。小数部分采用按权相加法。

    非十进制转非十进制

    以小数点为中心，往前数n位合成/拆分，往后数n位合成/拆分，不足的位数添零补足。

定点表示法：

    定点纯小数：1位符号位 + n位量值 例：1.0001，0.10101，实际值的绝对值一定小于1

    定点纯整数：1位符号位 + n位量值 例：10101，01111

    符号位为0是正数，符号位为0是负数

数的机器码表示（重灾区）：

int，float，double等，位数（长度）固定，都归类为机器码

机器码可分为原码、反码、补码、移码四种

原码：

    a.基本就是定点表示法，注意0有两种表示方式。

    b.特点：乘除简单，加减麻烦，为此引出补码，方便加减

补码：

    a.十进制数x转M进制下的补码值：`[x]补 =（x + M） mod M`

            所谓的补码值就是十进制下补码的值。

            从公式中可以看出，正数的补码就是正数本身，而负数的补码可以理解为（M-x）的原码（不考虑符号位）。比如M为12的情况下，x为2，则补码也为2；x为-10，则补码为（12-10）mod 12 = 2。

            考虑x=±6，M为12的情况下，它们的补码是一样的。为此，还需要一位符号位。这个符号位既表示符号，也参与计算。

            注意，补码中0只有一种表示方式，即0……000。

    b.补码与原码的转换

            从a中可以知道十进制数x转M进制补码值的方法，而想得到补码，其实只需做一件事：把补码值转换为对应进制下的原码。而补码与原码的转换，实际上就是跳过了求十进制值这一中间步骤。

            原码求补码，正数不变，**负数除符号位外取反加1**（也就是（M-x）的原码）。取反加1的过程也可以简化为从符号位到最后一个1中间的数位全部取反（不含两端），其他数位保持不变。至于为什么可以这样做，把M - x的过程转成二进制下的计算就清晰了。

            补码求原码，正数不变，负数就要从补码（M-x）中得到x。如果把（M - x）当做原码，我们要求的x又可以理解为（M-（M-x）），也就是求（M - x）的补码。所以很奇妙地，**方法又是除符号位外取反加1**。也就是说，补码和原码的双向转换方法完全是一样的。

            注意题目中提到n时，区分n指的是所有码位还是不含符号位的位数。

    c.双符号补码（变相补码），就是用双符号位11代替1，00代替0。
    
            在计算中如果结果的双符号位是01，表示发生了正溢出，也就是两个00加出了一个1，否则为负溢出。简单地计算几题就能理解了。

    d.特点：补码最大的优势就是能把减法变成加法，更改符号位即可。

反码：

    a.反码，顾名思义，就是把原码数位取反（除符号位）。取反的只有负数，正数依然保持不变。而补码是取反后加1，所以**如果原码是负数，补码一定比反码大1**。

    b.原码与补码的转换非常简单，不多赘述。补码与反码的转换也简单，±1即可。**反码中0有两种表示方式，全0和全1**。至于为什么会这样，现在我还不清楚。

移码：

    a.与补码除符号位外完全相同。由于补码的0只有一种，所以**移码的0也只有一种，也就是10……00**。

    b.移码常用于浮点数阶码中。

浮点表示法：

上述表示法中，往往只能表示纯整数或纯小数。为了表示一般实数，引入浮点表示法。

    a.浮点表示法中，小数点位置随阶码不同而浮动。书写格式为`N = R^E * M`。其中R为基数，表示进制。E为指数，M为尾数。在机器中，表示为（阶符）、阶码、（数符）、尾数。实际上可以简单地分为阶码E、尾数M两部分。前面只是表示符号而已。阶码E是带符号的定点整数，一般用移码和补码表示，决定**数的表示范围**。尾数M是带符号的定点小数，一般用原码和补码表示，决定**数的精度**。

    听着复杂，还是通过例子演示。十进制数26.5转换为二进制是11010.10，书写格式可以是+0.1101010 * 2^101，也可以是0.0110101*2^110。其实就是小数点前后挪一位的区别。但是假如浮点数格式都为5位移码+8位补码的情况下，前者的机器存储为10 101 0 1101010，后者是10 110 0 0110101。

    以10 101 0 1101010为例，这四部分分别为基数R 指数E 尾数M（0是符号位）。

    个人理解，这种表示方式类似于**科学计数法**。`N = R^E * M`本身形式就与科学计数法类似，完全可以类比学习。浮点数所谓的不唯一其实就是小数点标的位置不唯一，这就是“浮点”二字的由来。

    b.浮点数的规格化。其实就是科学计数法的规格化。不同的是，科学计数法一般尾数范围为1~10之间，即10^0~10^1，前开后闭。而浮点数的尾数范围是2^-1~2^0，前开后闭。负数的尾数范围与正数对称，不多赘述。

上述例子虽然简单，但是还没有体现“移码”和“补码”的转换过程，我们使用的一直是原码。

IEEE754标准中，规定尾数用原码，指数用移码。

4.1 指令系统的发展与性能要求

指令的地位：是硬件设计人员和软件设计人员之间的分界面，也是他们之间沟通的桥梁

（何意？）

指令系统基本概念：

指令：计算机执行某种操作的命令

从计算机组成的层次结构来说，指令可分为三种:

    宏指令：若干机器指令组成，属于软件

    **机器指令**：介于微指令和宏指令之间，每条指令可完成独立的算数逻辑或逻辑运算操作

    微指令：微程序级的命令，属于硬件

    本章重点讨论机器指令

指令系统：一台计算机所有机器指令的集合。

发展与性能要求：

    复杂指令系统计算机，简称CISC

    精简指令系统计算机，简称RISC

    性能要求：完备性、有效性、规整性、兼容性

4.2 指令格式：

    影响因素：机器字长、存储器容量、指令本身的功能

    反映信息：做什么操作、从哪获取操作数、结果送哪里、从哪获取下一条指令

具体而言，主要体现在两个方面：

a.操作码字段OP

表示该指令应进行什么性质的操作。操作码字段的位数一般取决于计算机指令系统的规模。

图片

图片

可以分为固定长度操作码、可变长度操作码。它们的区别要结合地址码来看。

b.地址码字段A

指示指令中的操作数的地址以及操作结果的存放地址。根据一条指令中有几个操作数地址，可将该指令称为几操作数指令或几地址指令。

三地址指令指令长度较长，一般只在字长较长的机器中使用。

二地址指令应用更为广泛。三地址指令中三个地址分别为操作数1、操作数2、操作结果，而二地址指令中第一个操作数的地址在运算结束后储存的不是第一个操作数，而是操作结果，也就是说A1的原存的内容已经被替换了。

    而SS、RS、RR是在二地址指令的基础上提出的。S表示存储器，R表示寄存器。它们表示操作数存放在硬件的哪个位置。

    如果要执行这条指令，SS要访问4次，第一次是读取指令，由于指令中没有直接给出操作数，而是给出操作数地址，所以要回到内存中取两次数据，最后一次得出操作结果，要把结果放回内存中，又要读取一次。总共是四次。

    RS要访问2~3次。读取指令都需要一次，但是读取操作数只需要一次（另一次直接在寄存器中读取），这里已经是2次了。关键在于最后一次。由于二地址指令的操作结果是要覆盖操作数的，如果第一个操作数存放在内存中，那么存放结果的时候还要回到内存，所以又加一次。这种情况下是3次。反之如果第一个操作数放在寄存器里，那就不用回去读内存，这时总共只需访问2次内存。

    RR只需一次。没什么好说的，读取指令访问一次就行了。其他操作都和存储器无关。

一地址指令稍微有些不同。它只能执行单操作数指令，如自增、自减等。

c.指令字长度

机器字长是计算机能直接处理的二进制数据位数。而对应的指令字长就是指令中包含二进制代码的位数。根据机器字长和指令字长长度的关系，指令长度可以是单字长指令（等于机器字长）、半字长指令（等于0.5机器字长）、双字长指令（等于2机器字长）等。

多字长指令：

    优点：有足够的地址位解决访问内存任何单元的寻址问题。也就是地址位够多，可以随便取数据。

    缺点：其他指令访问内存一次就能取出来，但是它要取多次，而且还占空间。

等长指令字结构：各指令长度相等（单字/半字）

    优点：指令字结构简单，而且长度整齐不变。

变长指令字结构：长度随指令功能而异

    优点：能充分利用指令长度，但是指令的控制很复杂。

    2.确定下一条指令的地址

寻址方式：

    指CPU根据OP和A，形成操作数或指令地址的方式。

指令寻址和数据寻址又是何意？

指令的寻址方式：

    顺序寻址方式：PC+1，其中PC为程序计数器（指令指针寄存器）

    跳跃寻址方式（条件转移和无条件转移指令）：
            类似goto语句。
            下条指令的地址不是PC给的，而是本指令给的。
            当然，跳转后PC要调整为程序的内存地址。

操作数的寻址方式：

    操作数来源：

            指令地址码直接给出。方便快捷，但操作数固定不变。

            放在CPU的通用数据寄存器中，快速，但是存储草桌数数量有限。

            放在内存的数据区中：有效地址、形式地址（二维指针）。
