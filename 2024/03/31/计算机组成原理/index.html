第二章：运算方法和运算器

2.1 数字与文字的表示方法

进制转换：

十进制转非十进制

整数部分较为简单。小数部分采用乘基取整法。

非十进制转十进制

整数部分较为简单。小数部分采用按权相加法。

非十进制转非十进制

以小数点为中心，往前数n位合成/拆分，往后数n位合成/拆分，不足的位数添零补足。

定点表示法：

定点纯小数：1位符号位 + n位量值 例：1.0001，0.10101，实际值的绝对值一定小于1
定点纯整数：1位符号位 + n位量值 例：10101，01111

符号位为0是正数，符号位为0是负数

数的机器码表示（重灾区）：

int，float，double等，位数（长度）固定，都归类为机器码

机器码可分为原码、反码、补码、移码四种

原码：

a.基本就是定点表示法，注意0有两种表示方式。

b.特点：乘除简单，加减麻烦，为此引出补码，方便加减

补码：

a.十进制数x转M进制下的补码值：[x]补 =（x + M） mod M

所谓的补码值就是十进制下补码的值。

从公式中可以看出，正数的补码就是正数本身，而负数的补码可以理解为（M-x）的原码（不考虑符号位）。比如M为12的情况下，x为2，则补码也为2；x为-10，则补码为（12-10）mod 12 = 2。

考虑x=±6，M为12的情况下，它们的补码是一样的。为此，还需要一位符号位。这个符号位既表示符号，也参与计算。

注意，补码中0只有一种表示方式，即0……000。

b.补码与原码的转换

从a中可以知道十进制数x转M进制补码值的方法，而想得到补码，其实只需做一件事：把补码值转换为对应进制下的原码。而补码与原码的转换，实际上就是跳过了求十进制值这一中间步骤。

原码求补码，正数不变，负数除符号位外取反加1（也就是（M-x）的原码）。取反加1的过程也可以简化为从符号位到最后一个1中间的数位全部取反（不含两端），其他数位保持不变。至于为什么可以这样做，把M - x的过程转成二进制下的计算就清晰了。

补码求原码，正数不变，负数就要从补码（M-x）中得到x。如果把（M - x）当做原码，我们要求的x又可以理解为（M-（M-x）），也就是求（M - x）的补码。所以很奇妙地，方法又是除符号位外取反加1。也就是说，补码和原码的双向转换方法完全是一样的。

注意题目中提到n时，区分n指的是所有码位还是不含符号位的位数。

c.双符号补码（变相补码），就是用双符号位11代替1，00代替0。在计算中如果结果的双符号位是01，表示发生了正溢出，也就是两个00加出了一个1，否则为负溢出。简单地计算几题就能理解了。

d.特点：补码最大的优势就是能把减法变成加法，更改符号位即可。

反码：

a.反码，顾名思义，就是把原码数位取反（除符号位）。取反的只有负数，正数依然保持不变。而补码是取反后加1，所以如果原码是负数，补码一定比反码大1。

b.原码与补码的转换非常简单，不多赘述。补码与反码的转换也简单，±1即可。反码中0有两种表示方式，全0和全1。至于为什么会这样，现在我还不清楚。

移码：

a.与补码除符号位外完全相同。由于补码的0只有一种，所以移码的0也只有一种，也就是10……00。

b.移码常用于浮点数阶码中。

浮点表示法：

上述表示法中，往往只能表示纯整数或纯小数。为了表示一般实数，引入浮点表示法。

a.浮点表示法中，小数点位置随阶码不同而浮动。书写格式为N = R^E * M。其中R为基数，表示进制。E为指数，M为尾数。在机器中，表示为（阶符）、阶码、（数符）、尾数。实际上可以简单地分为阶码E、尾数M两部分。前面只是表示符号而已。阶码E是带符号的定点整数，一般用移码和补码表示，决定数的表示范围。尾数M是带符号的定点小数，一般用原码和补码表示，决定数的精度。

听着复杂，还是通过例子演示。十进制数26.5转换为二进制是11010.10，书写格式可以是+0.1101010 * 2^101，也可以是0.0110101*2^110。其实就是小数点前后挪一位的区别。但是假如浮点数格式都为5位移码+8位补码的情况下，前者的机器存储为10 101 0 1101010，后者是10 110 0 0110101。

以10 101 0 1101010为例，这四部分分别为基数R 指数E 尾数M（0是符号位）。

个人理解，这种表示方式类似于科学计数法。N = R^E * M本身形式就与科学计数法类似，完全可以类比学习。浮点数所谓的不唯一其实就是小数点标的位置不唯一，这就是“浮点”二字的由来。

b.浮点数的规格化。其实就是科学计数法的规格化。不同的是，科学计数法一般尾数范围为110之间，即10^010^1，前开后闭。而浮点数的尾数范围是2^-1~2^0，前开后闭。负数的尾数范围与正数对称，不多赘述。

上述例子虽然简单，但是还没有体现“移码”和“补码”的转换过程，我们使用的一直是原码。

IEEE754标准中，规定尾数用原码，指数用移码。

第三章 内部存储器

3.1 存储器概述

存储可以按许多标准分类。其中需要了解的标准：

按存取时间和存取位置关系，依次可以分为随机存取存储器（RAM），顺序存取存储器（SAM），半顺序存取存储器（直接存取存储器DAM）；

判断标准理解为，存储器的任何一个存储单元的内容是否都可以随机存取，而且存取时间与存储单元的物理位置是否有关。

按读写功能分类，可以分为RAM：双极型/MOS型（SRAM/DRAM），ROM：MROM/PROM/EPROM/EEPROM/Flash Memory。

RAM与ROM最大的区别就是读写功能。

判断标准理解为，存储器是可写还是只读，以及写入次数等。

此外还有按信息的可保存性分类，与CPU的耦合程度等。

二级储存系统：

把存储器简单地分为内存和外存。其中内存为内寄存器（也称主存），CPU可直接访问；外存为辅存，CPU不能直接处理。

三级存储系统：

在主存与CPU之间，增加一种新的存储器：高速缓冲存储器（Cache）。Cache和主存都能直接与CPU交换数据。增加Cache以后，三级储存系统速度接近Cache，但容量和价格接近外存，提高了整个存储系统的性价比。

多级存储系统：

有多级Cache分层。

分层存储器系统之间的连接关系：

程序局部性原理（构成多级存储系统的依据）：

a.时间局部性：最近被访问的信息很可能还要被访问

b.空间局部性：最近被访问的信息邻近地址的信息也可能被访问

主存内容是辅存内容的拷贝；Cache内容是主存内容的拷贝。这句话个人理解为主存内容是辅存内容的一部分；Cache内容是主存内容的一部分。

内存的编址单元

1.字节存储单元：存放一个字节的单元，相应的地址称为字节地址

2.字存储单元：存放一个机器字的存储单元，相应的单元地址叫字地址

编址单元（以上两者单元）相同，主存容量越大，地址码越长；主存容量相同，编址单位越小，地址码越长

如何理解？当主存容量越大时，需要更多的地址码位数来表示不同的编址单元，这意味着地址码越长；编址单位越小，即每个编址单元存储的数据越少，那么编址单元的数量也会相应增加，而每个编址单元都需要一个地址码来唯一标识，所以当编址单位变小时，需要更多的地址码位数来表示不同的编址单元，这也意味着地址码越长。

主要性能指标：储存容量、存取时间、存储周期、存储器带宽

3.2 SRAM存储器

主存是半导体存储器

根据信息存储的机理不同，主存可以分为两类：

1.静态读写储存器（SRAM）：存取速度快

2.动态读写存储器（DRAM）：存储容量大

SRAM存储器

定义：是一种基本的静态存储元阵列

其中存储元是指锁存器（触发器），掉电丢失。SRAM存储器由存储元阵列构成。也就是说，存储元是SRAM的基本单位，而SRAM中的存储元是触发器。

触发器：由4个MOS管组成，2个MOS管开关负责连接x和y向选择线，2个MOS管负责状态切换。

三组信号线：

1.地址线：指定要读取或写入的存储单元的位置

2.数据线：用于传输读取或写入的数据，其宽度由存储器的字长确定。可以分为行线和列线

3.控制线：读写操作不同时发生

因此三组信号线工作的流程为寻址、操作数据（读写不会同时发生）。

3.3 DRAM存储器

定义：和SRAM存储器一样，DRAM也是存储元列阵。不同之处在于它是动态的，且存储元是记忆电路。

记忆电路是由一个MOS管和电容器组成的。

区别：静态指的是SRAM使用的是静态存储单元，每个存储单元由一个触发器构成，可以保持数据的稳定状态，即只要电源供应不中断，数据就会持续保存，不需要刷新。因此，SRAM访问速度快，但相对占用更多的面积并且功耗较高。

动态指的是DRAM使用的是动态存储单元，每个存储单元是由一个存储电容和一个访问晶体管组成。电容作为存储单元，需要定期刷新以保持数据的稳定性，所以被称为动态，因为数据不断被刷新。DRAM具有较高的存储密度和较低的功耗，但相对于SRAM而言，访问速度较慢。

存储器容量的扩充

1.位扩展 用给定的几块位数较短的芯片，拼一块位数较大的芯片

什么是芯片的位数？芯片的位数描述了其数据总线或寄存器的宽度，其实就是芯片能够一次处理的二进制位数。

说得再拟人一点，就是芯片处理的基本单位。

例2：利用1K×4位的SRAM芯片，设计一个存储容量为1K×8位的SRAM存储器

解：所需芯片数量d = (1K×8)/(1K×4)=2片

2.字扩展 用给定的几块字数较短的芯片，拼一块字数较大的芯片

字数又是什么？字数描述了处理器能够一次操作的字的个数。跟位数还是有区别的。

假设有一个16位的处理器，它的字长是16位，即一次能够处理16位的数据。在这个处理器中，一个字可以由两个16位的数据组成，因此字数为2。换句话说，这个处理器的字数是2，但每个字包含16位，因此整个处理器有32位（16位 x 2个字），位数为32。

我的理解是字数是作业本，位数是构成作业本的一页纸。你可以拿一捆作业本批量处理，但是具体到处理某一个本子时，肯定是一张一张翻着看的。
