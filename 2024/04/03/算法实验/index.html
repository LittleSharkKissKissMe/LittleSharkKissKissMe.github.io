题目：动态规划求解数组连续最大和

1.CSDN思路

子问题界定：前边界为1，后边界为i，i从n往前推

这里有两个重点：

1、双边界中，我们只调整一个边界，固定另一个边界

2、调整的边界往往是从后往前推的

对于第二点，我的理解是所有子问题的最优解之和不一定是全局最优解，但全局最优解一定包含着若干局部最优解。

参考链接：https://blog.csdn.net/DCXY71/article/details/108388987

对于第一点，别急，接着往下看。

所以假设C[i]就是当问题界定为1到i之间时，计算出来的子序列最大和，那么可以递推：C[i]=max(C[i-1],C[i-1]+num[i])?

1、不选num[i]。也就是num[i]的存在无关紧要，

2、选择num[i]。那么情况就复杂了？可以是各种奇奇怪怪的分割，也就是说即使选择了num[i]，C[i-1]连续的序列也不一定刚好就能连到num[i-1]，万一人家是前面某一段呢？只是num[i]太大导致C[i]变成后面的那一段？

然而，对……对吗？（半知半解龙.jpg）

这里我的思维是考虑边界的取舍，进而缩小边界，这样要考虑复杂的情况。

而用动态规划的思维其实非常粗暴：C[i]=max(num[i],C[i-1]+num[i])

解就是max(C[1],C[2],……C[i])

但是直接理解这个式子可能还是有点不知所谓了。

举个例子：1,2,3,4,-999,20

这里最大和显然是20。

假如末尾恰好停留在20，那么前面五个数的连续子数组的最大和是10，nums[i]是20，按照递推公式10+20>20，最大和应该是30，但是我们都知道它们中间隔了一个-999，根本不连续，取不到30。

何意啊？

实际上，max比较的两边都含有num[i]，所以max里比较的无非是C[i-1]是正是负。假如我们计算C[i-1]，C[i-1]为-989，实际上小于0，是要被舍去的。这又是如何做到的？

对于这个式子，我们要知道，C[i]实际上不是1到i之间的连续子数组的最大和，如1,2,3,4，而是以末尾元素结尾的连续子数组的最大和，如1,2,3,4,-999。MAX比较的实际意义是，由于前面肯定是连到num[i]的，我只要判断要不要让你连过来。

也就是说，在i的遍历中，后边界在不断缩小，而前边界是动态规划的最优解确定的，1不是前边界，仅仅界定了一个可供选择的范围。

最后的解只要从诸多C[i]中取最大即可。也就是max{C[i]}，i从1到n取。

何意？天意！乍一看不明所以，实则恍然大悟。是的，它在缩边界，但是缩的时候又保证每种情况都考虑到，不同于暴力双层遍历，后边界i的遍历确实是全遍历，但前边界j只遍历了最优的情况，而每个确定的边界下最优情况只有一种，可以理解为其实对于每个i，j其实就只有那么一种情况，也就是子问题得出的最优情况。在每个i的遍历中，你只要判断max是哪个就行。

所以时间复杂度为O(n*1)=O(n)。

至于为什么j能确定最优解，对于一般的动态规划而言，人家有备忘录，可以直接记下，所以不需要像蛮子一样每个后边界再遍历一次前边界。

悟了，这下真悟了。

固一优二，大抵就是今天悟出来的精髓了。

虽然但是，这是为了方便构造最优子结构而假设的，实际解决动态规划的时候只要能构造出最优子结构解决问题就行了，没必要硬凑固一优二。

但是显然还没有结束，我们还要把想法付诸实践，也就是写出相应的代码。

虽然不知道时间够不够，还是先考虑解决一个问题，再递推到其他问题

由伪代码，

核心思想：C[i]=max(num[i],C[i-1]+num[i])

int MaxSum(int* A,int n){
int sum=0;
int b=0; //b就是C[i]，只不过每次循环都比较一次，所以不需要用数组储存，只需用一个变量即可。
int c=n;

for(int i=0;i<n;i++){   //伟大的单轮循环

    if(b>0) b+=A[i];    //这段if相当于C[i]=max(num[i],C[i-1]+num[i])
    else b=A[i];

    if(b>sum){          //这段if相当于选出最大的C[i]
        sum=b;

        c=i;            //c是子段最后位置
    }
}
return sum,c;

}

但是个人感觉这个例子并没有显示备忘录的作用？看下一个例子罢

另外还有一个大问题，为什么是固定到以末尾元素结尾的连续子数组？这一点是怎么想到的？如果没有这个条件，实际计算会复杂得多。

答：

"以当前元素为结尾的最大子数组的和"这个概念来源于动态规划中最优子结构的思想。在解决连续最大和问题时，我们可以通过定义这样一个状态来辅助我们思考和设计算法。"

也就是说，只要我定义的**辅助状态**是可以**覆盖到全部情况**的，那么我就可以直接假设这个状态吗？

"是的，正确理解了！在动态规划中，我们通常会定义一些辅助状态来帮助解决问题。当我们能够确保所定义的辅助状态可以覆盖到全部情况，并且能够通过这些状态推导出最终结果时，我们就可以直接假设这些状态，并根据状态之间的关系设计出相应的状态转移方程。

通过定义合适的辅助状态，我们可以简化问题的复杂度，将问题分解成更容易处理的子问题。这样不仅能够更好地理解问题，还能够设计出高效的算法来解决问题。在使用动态规划求解问题时，合理定义辅助状态是非常重要的一步。"

暴力算法略，按照老套路，采用固一优二的方案

从后往前，缩短后边界

2、前面是以末尾元素结尾的连续子序列的最大和，但是i位置的字母不一定是公共的，如何作结尾？

也就是说，用双后边界，i和j同时对着两个序列遍历

此时又要多想一步：刚刚是固定后边界，也就是确保遍历的后边界一定是最长公共子序列的后边界（这里表达有点乱，实际意思就是前面找的公共子序列一定要以遍历中的后边界为后边界，参考示例1），现在后边界不一定是公共的，如何固定？

答案是分类讨论。简单的方法，极致的效率。

实际上，子序列的后边界无非是三种情况：等于数列A后边界，等于数列B后边界，都等于，都不等于

而且在我们的假设中，已经给定了这个后边界不管属于A还是B，一定是在最长公共子序列里的，所以不可能都不等于，最后一种情况可以直接不管。
