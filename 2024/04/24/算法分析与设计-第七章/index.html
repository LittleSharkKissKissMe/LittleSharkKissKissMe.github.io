第四章 贪心算法

4.1 贪心算法简介

定义：

贪心算法（greedy algorithm，又称贪婪算法）是指，在对问题求解时，总是做出在当前看来是最好的选择，即贪心算法并不从整体最优上加以考虑，它所得出的仅仅是在某种意义上的局部最优解。

区别（与动态规划相比）：

1.解决问题的性质。贪心算法适用于的问题通常是在每个决策步骤中，当前的最优选择对未来步骤的影响是可以预测的，例如，寻找最短路径问题时，总是选择当前看起来最短的路径；动态规划适用于子问题重叠的情况，每个子问题的解决方案可以被缓存并用于解决其他相似的子问题，例如，在背包问题中，可能会有多个子问题需要解决，而这些子问题之间存在重叠。

2.算法的优缺点。贪心算法通常较为简单，因此在计算上更高效，但由于它是一种局部搜索算法，可能无法找到全局最优解；动态规划则能保证找到全局最优解，但通常实现起来更复杂，计算量也更大。

3.适用的问题类型。贪心算法适用于那些可以分解为独立步骤的问题，每一步的选择只依赖于当前的状态，而不需要回溯或考虑未来的步骤；动态规划则适用于那些可以分解为相互依赖的子问题的情况，其中子问题的解决方案可以被重复利用以避免重复计算。

后面将通过实例具体解释。

单源最短路径问题

问题描述：离散数学典型问题，直接上代码

代码：

    void Dijkstra(int n, int v, int *dist, int *prev, int **c) {

        bool s[maxint];    // 判断是否已存入该点到S集合中

        //初始化
        for(int i=1; i<=n; ++i){
            dist[i] = c[v][i];
            s[i] = 0;     // 所有点都未加入集合S

            if(dist[i] == maxint)
                prev[i] = 0;
            else
                prev[i] = v;
        }

        dist[v] = 0;    
        s[v] = 1;

        for(int i=2; i<=n; ++i){
            int tmp = maxint, u = v;
                   
            // 找出当前未使用的点j的dist[j]最小值
            for(int j=1; j<=n; ++j)
                if((!s[j]) && dist[j]<tmp){ //点j未加入集合S，且j到起始节点v的当前最短距离小于之前找到的最小距离tmp
                    u = j; tmp = dist[j];
                }

            s[u] = 1;    // 表示u点已加入S集合

            for(int j=1; j<=n; ++j)  // 更新dist
                if((!s[j]) && c[u][j]<maxint){  //点j未加入集合S，且u-j是一条边，则重新计算dist，并更新前驱节点

                    int newdist = dist[u] + c[u][j];

                    if(newdist < dist[j]){
                        dist[j] = newdist;  prev[j] = u;
                    }
                }
        }
    }

    说明：
    1) n为顶点数，顶点v为源；
    2) c[i][j]表示边(i,j)的权；当c[i][j]为大数时，它不是一条边。
    3) dist[i]表示从v到i的最短特殊路径长度。

    代码主要做以下几件事：
        1.从第2个节点开始遍历到第n个节点，遍历过程中，每次选择距离最短的节点加入最短路径节点集合S。
        2.找出当前未使用的节点u的dist[u]值最小的节点，并将其加入最短路径节点集合S。
        **3.在未加入最短路径节点集合S中的节点中，更新节点到该节点的距离，并更新该节点的前驱节点。**

优先队列实现代码：

    #include <iostream>
    #include <vector>
    #include <queue>
    #include <limits.h>

    using namespace std;

    #define maxint INT_MAX

    void Dijkstra(int n, int v, vector<int>& dist, vector<int>& prev, vector<vector<int>>& c) {
        priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq; // 优先队列，按照距离排序
        //pair对象实际上是一个有序的容器，可以存储两个值，可以是相同类型或不同类型。
        //这句代码创建了一个优先队列对象pq，元素类型是pair<int, int>，并使用greater<pair<int, int>>作为排序准则，表示距离值小的元素优先级更高。
        //greater从小到大排序，pair中会以第一个数为基准

        dist.assign(n + 1, maxint);     //连续赋值，初始化dist
        prev.assign(n + 1, 0);          //连续赋值，初始化prev
        dist[v] = 0;

        pq.push(make_pair(0, v));

        while (!pq.empty()) {
            int u = pq.top().second;
            pq.pop();

            for (int j = 1; j <= n; ++j) {
                if (c[u][j] < maxint) {
                    int newdist = dist[u] + c[u][j];
                    if (newdist < dist[j]) {
                        dist[j] = newdist;
                        prev[j] = u;
                        pq.push(make_pair(dist[j], j)); // 更新距离后将节点加入优先队列
                    }
                }
            }
        }
    }

    int main() {
        int n = 4; // 图中节点数
        int v = 1; // 起始节点
        vector<int> dist(n + 1);
        vector<int> prev(n + 1);
        vector<vector<int>> c = {
            {maxint, maxint, maxint, maxint, maxint},
            {maxint, 0, 1, 2, maxint},
            {maxint, maxint, 0, maxint, 3},
            {maxint, maxint, maxint, 0, 1},
            {maxint, maxint, maxint, maxint, 0}
        };

        Dijkstra(n, v, dist, prev, c);

        for (int i = 1; i <= n; ++i) {
            cout << "Minimum distance from node " << v << " to node " << i << " is: " << dist[i] << endl;
        }

        return 0;
    }

数学归纳法

假设F(n-1)满足目标假设，用F(n-1)证明F(n)，同时证明F(1)也满足目标假设。即可证明假设成立

第一数学归纳法

适合证明涉及自然数的命题P(n)。

归纳基础：证明P(1)为真（或P(0)为真）

归纳步骤：若对所有n有P(n)为真，证明P(n+1)为真

第二数学归纳法

适合证明涉及自然数的命题 P(n)。

归纳基础：证明P(1)为真（或P(0)为真）

归纳步骤：若对所有小于n的k有P(k)真，证明 P(n)为真
（何意啊？？？）

两种归纳法区别：

图片

在第一数学归纳法中，我们假设命题在 n=k 时成立，并通过这个假设推导出命题在 n=k+1 时成立。而在第二数学归纳法中，我们假设命题在 n=1,2,...,k 都成立，并通过这些假设推导出命题在 n=k+1 时成立。因此，第二数学归纳法使用更多的前提条件，从而在一些情况下比第一数学归纳法更为强大。

4.4 最优装载问题

问题：n 个集装箱1, 2, … , n 装上轮船，集装箱 i 的重量 wi , 轮船装载重量限制为C, 无体积限制。问如何装使得上船的集装箱最多？不妨设每个箱子的重量 wi ≤ C。

（实际就是每种集装箱只有1个，要在满足限重的情况下装载尽可能多的箱子）

（相当于0-1背包问题）

建模：设 <x1, x2, ... , xn> 表示解向量，xi= 0,1。其中xi = 1表示当且仅当第 i 个集装箱装上船。

目标函数：max Xi之和，其中i范围为1-n。（也就是求箱子最多能装几个）

约束条件：WiXi之和 <= C，其中其中i范围为1-n。（满足限重）
